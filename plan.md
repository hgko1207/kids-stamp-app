## 다음 작업 (구현 예정)

### 1. 도장 부정 찍기 방지 시스템 ⭐ (신규)

#### 문제 상황
아이가 혼자 패드를 들고 어제/그제 도장을 몰래 추가로 찍을 수 있음.

#### 타 앱 분석

| 앱 | 방식 | 특징 |
|----|------|------|
| **ClassDojo** | 선생님/부모만 점수 부여 | 아이는 보기만 가능 |
| **iReward Chart** | PIN 잠금 후 부모가 찍어줌 | PIN 입력 필수 |
| **Chore Monster** | 아이가 "요청" → 부모 승인 | 투명한 확인 과정 |
| **OurHome** | 부모 승인 + 기록 남김 | 가족 공유 |
| **Habitica** | 본인 책임 자율 입력 | 성인 대상이라 자율 |

#### 추천 방식 — 3가지 옵션 비교

**옵션 A: 오늘 날짜만 찍기 허용** ⭐ 추천
- 오늘 날짜 칸만 활성화, 과거/미래는 터치 불가
- 장점: 구현 심플, 아이 흐름 방해 없음
- 단점: 부모가 깜빡하고 못 찍어줬을 때 소급 불가
- 해결: 부모 설정에서 "잠금 해제" 토글로 과거 날짜 수정 가능하게

**옵션 B: 하루 최대 N개 제한**
- 하루에 1개만 찍을 수 있도록 제한
- 장점: 자연스럽게 과도한 찍기 방지
- 단점: 여러 칭찬을 한 날에 주고 싶을 때 불편

**옵션 C: 도장 찍기 PIN 인증**
- 도장 찍기 전에 4자리 PIN 입력 필요
- 장점: 완벽한 방지
- 단점: 매번 PIN 입력 → 아이가 흥미 잃을 수 있음, 부모도 번거로움

#### 최종 설계 — 옵션 A 구현 (부모 해제 가능)

```
부모 설정에 "도장 잠금 모드" 토글 추가

[잠금 ON] (기본값)
  - 오늘 날짜만 찍기 가능
  - 과거 날짜: 도장 표시는 보이지만 터치 비활성화
  - 미래 날짜: 비활성화

[잠금 OFF] (부모가 해제)
  - 모든 날짜 자유롭게 찍기/지우기 가능
  - 소급 수정, 실수 정정 시 사용
```

- **구현 파일**: `src/types.ts`에 `stampLock: boolean` 필드 추가
- **변경 파일**: `Calendar.tsx` — 잠금 시 오늘 외 날짜 `pointer-events-none` + 시각적 흐림 처리
- **변경 파일**: `ParentSettings.tsx` — 잠금 모드 토글 추가
